# Security Policy

## Supported Versions

AniFlux is actively developed. Security updates apply only to the latest
released version available on GitHub and the Google Play Store.

| Version | Supported |
| ------- | --------- |
| Latest  | ✅ Yes    |
| Older   | ❌ No     |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### How to Report
- **Do NOT** open a public GitHub issue for security vulnerabilities.
- Report the issue privately using one of the following methods:
  - GitHub Security Advisories (preferred)
  - Or open a GitHub issue **clearly marked as security-related** with minimal details

### What to Include
Please provide:
- A clear description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Screenshots or logs (if safe to share)

---

## Response Expectations

- You can expect an initial response within **7 days**
- If the issue is confirmed, a fix will be worked on as soon as possible
- Once resolved, the fix may be publicly disclosed

---

## Scope

This security policy applies to:
- The AniFlux application source code
- Build and release configurations
- Data handling within the app

This policy does **not** apply to:
- AniList services or infrastructure
- Third-party libraries or APIs (report those to their maintainers)

---

## Responsible Disclosure

Please avoid:
- Exploiting vulnerabilities
- Accessing user data without permission
- Public disclosure before a fix is released

Responsible disclosure helps keep users safe and the project healthy.

---

Thank you for helping keep AniFlux secure.
